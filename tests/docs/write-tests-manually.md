# How to hand-write tests by importing `@kiwa-test/core` as a library

> [🇬🇧 English](./write-tests-manually.md) • [🇯🇵 日本語](./write-tests-manually.ja.md)

This is the route where you do **not** use skills such as `/kiwa-test` or `/kiwa-play`, but instead import `@kiwa-test/core` directly as an npm library and write the tests yourself. This is for use cases like integrating kiwa into an existing dApp project as a library, preferring hand-written tests because skill-generated tests are harder to read, or reusing only selected fixtures.

If you want full auto-generation from zero through the skill path, read [run-tests.md](./run-tests.md).

## When to choose direct library usage

| Situation | Recommended path |
|---|---|
| Introduce kiwa into a new dApp and auto-generate tests from zero | Skill path ([run-tests.md](./run-tests.md)) |
| Add tests to an existing dApp and let kiwa's AI design the viewpoints / structure | Skill path ([retrofit-existing-dapp.md](./retrofit-existing-dapp.md)) |
| Add tests to an existing dApp **by hand**, reusing only kiwa fixtures / helpers | **This doc (direct library usage)** |
| Read and replace skill-generated tests, or swap only some helpers over to kiwa | This doc |
| Reuse only the anvil startup helper in an existing Hardhat / Foundry project | This doc (import only `startAnvil` / `startAnvilCluster`) |

## Step 0 — Prerequisites + install

Run this at the root of your own dApp project.

```bash
# Install @kiwa-test/core (main deps)
pnpm add -D @kiwa-test/core @playwright/test viem

# Foundry (needed to launch anvil) must be on PATH
anvil --version

# Playwright chromium
pnpm exec playwright install chromium
```

Set `"type"` in `package.json` to `"module"` (`kiwa-core` is ESM).

## Step 1 — Write `playwright.config.ts`

The `@kiwa-test/core` fixture extends Playwright's standard `test`, so you can keep using a normal Playwright config.

```typescript
// playwright.config.ts
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  timeout: 30_000,
  fullyParallel: false,    // parallel runs are disabled because anvil is shared
  reporter: 'list',
  use: {
    headless: true,
  },
  projects: [{ name: 'chromium', use: { ...devices['Desktop Chrome'] } }],
});
```

## Step 2 — Launch anvil + deploy contracts in `tests/prepare-env.ts`

`@kiwa-test/core`'s `runE2EPrepareEnv` wraps anvil child-process startup, contract deployment, and `.env.local` output into one function.

```typescript
// tests/prepare-env.ts
import { runE2EPrepareEnv, loadForgeArtifact } from '@kiwa-test/core';
import { resolve, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const exampleRoot = resolve(__dirname, '..');

await runE2EPrepareEnv({
  envFile: resolve(exampleRoot, '.env.local'),
  port: 8551,    // a port that does not collide with other examples
  deploy: async ({ wallet, publicClient }) => {
    // Read the artifact generated by forge build
    const artifact = loadForgeArtifact({
      path: resolve(exampleRoot, 'forge-out/MintNft.sol/MintNft.json'),
    });
    const hash = await wallet.deployContract({
      abi: artifact.abi,
      bytecode: artifact.bytecode.object,
      args: ['MyNft', 'MNFT'],
    });
    const receipt = await publicClient.waitForTransactionReceipt({ hash });
    return {
      NEXT_PUBLIC_MINT_NFT: receipt.contractAddress!,
    };
  },
});
```

`runE2EPrepareEnv` is responsible for:

- securing a port via `getFreePort` (automatically switching if the requested port collides)
- launching an anvil child process via `startAnvil`
- passing wallet clients into the `deploy` callback so you can deploy contracts
- writing deployed addresses into `.env.local`
- automatically stopping anvil when the process exits

## Step 3 — Hand-write `tests/{module}.spec.ts`

Import the `dappE2eTest` fixture and write Playwright tests. The fixture injects `anvilPort` automatically.

### Minimal sample (`mint-nft`, 1 contract, happy path)

```typescript
// tests/mint.spec.ts
import { test, expect } from '@playwright/test';
import { dappE2eTest } from '@kiwa-test/core';
import { createPublicClient, createWalletClient, http, defineChain, type Address } from 'viem';
import { privateKeyToAccount } from 'viem/accounts';
import { readFileSync } from 'node:fs';
import { resolve, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const exampleRoot = resolve(__dirname, '..');

// Wallet for anvil (fixed PK, test-only)
const MINTER_PK = '0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80' as const;

// Contract ABI
const nftArtifact = JSON.parse(
  readFileSync(resolve(exampleRoot, 'forge-out/MintNft.sol/MintNft.json'), 'utf8')
);

dappE2eTest('TC-001 mint increases balance by 1', async ({ anvilPort, page }) => {
  const chain = defineChain({
    id: 31337,
    name: 'anvil',
    nativeCurrency: { name: 'Ether', symbol: 'ETH', decimals: 18 },
    rpcUrls: { default: { http: [`http://127.0.0.1:${anvilPort}`] } },
  });

  const publicClient = createPublicClient({ chain, transport: http() });
  const walletClient = createWalletClient({
    chain,
    transport: http(),
    account: privateKeyToAccount(MINTER_PK),
  });

  const nftAddress = process.env.NEXT_PUBLIC_MINT_NFT as Address;

  // Action: click the mint button through the UI -> sign the wallet tx -> send tx
  await page.goto('http://127.0.0.1:3000');
  await page.getByTestId('connect-button').click();
  await page.getByTestId('mint-button').click();

  // Expectation: balance changes from 0 -> 1
  const balance = await publicClient.readContract({
    address: nftAddress,
    abi: nftArtifact.abi,
    functionName: 'balanceOf',
    args: [walletClient.account.address],
  });

  expect(balance).toBe(1n);
});
```

### Multi-contract interaction sample (`nft-marketplace`, mint -> list -> buy)

```typescript
// tests/marketplace.spec.ts
import { dappE2eTest } from '@kiwa-test/core';
import { expect } from '@playwright/test';
import {
  createPublicClient, createWalletClient, http, defineChain,
  parseEther, type Address,
} from 'viem';
import { privateKeyToAccount } from 'viem/accounts';
import { readFileSync } from 'node:fs';
import { resolve, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const exampleRoot = resolve(__dirname, '..');

const SELLER_PK = '0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80' as const;
const BUYER_PK  = '0x59c6995e998f97a5a0044966f0945389dc9e86dae88c7a8412f4603b6b78690d' as const;

const nftArtifact = JSON.parse(readFileSync(resolve(exampleRoot, 'forge-out/MarketNft.sol/MarketNft.json'), 'utf8'));
const marketArtifact = JSON.parse(readFileSync(resolve(exampleRoot, 'forge-out/SimpleMarketplace.sol/SimpleMarketplace.json'), 'utf8'));

dappE2eTest('TC-001 owner moves from seller to buyer after mint -> list -> buy', async ({ anvilPort }) => {
  const chain = defineChain({
    id: 31337, name: 'anvil',
    nativeCurrency: { name: 'ETH', symbol: 'ETH', decimals: 18 },
    rpcUrls: { default: { http: [`http://127.0.0.1:${anvilPort}`] } },
  });
  const publicClient = createPublicClient({ chain, transport: http() });
  const seller = createWalletClient({ chain, transport: http(), account: privateKeyToAccount(SELLER_PK) });
  const buyer = createWalletClient({ chain, transport: http(), account: privateKeyToAccount(BUYER_PK) });

  const nftAddr = process.env.NEXT_PUBLIC_NFT as Address;
  const marketAddr = process.env.NEXT_PUBLIC_MARKETPLACE as Address;

  // 1. seller mints the NFT
  const mintHash = await seller.writeContract({
    address: nftAddr, abi: nftArtifact.abi,
    functionName: 'mint', args: [seller.account.address],
  });
  await publicClient.waitForTransactionReceipt({ hash: mintHash });

  // 2. seller approves the NFT for the marketplace
  await seller.writeContract({
    address: nftAddr, abi: nftArtifact.abi,
    functionName: 'setApprovalForAll', args: [marketAddr, true],
  });

  // 3. seller lists the NFT
  await seller.writeContract({
    address: marketAddr, abi: marketArtifact.abi,
    functionName: 'list', args: [1n, parseEther('1')],
  });

  // 4. buyer buys it (paying 1 ETH)
  const buyHash = await buyer.writeContract({
    address: marketAddr, abi: marketArtifact.abi,
    functionName: 'buy', args: [1n], value: parseEther('1'),
  });
  await publicClient.waitForTransactionReceipt({ hash: buyHash });

  // 5. Expectation: NFT owner is now buyer
  const owner = await publicClient.readContract({
    address: nftAddr, abi: nftArtifact.abi,
    functionName: 'ownerOf', args: [1n],
  });
  expect(owner).toBe(buyer.account.address);
});
```

### Isolating tests with snapshot / revert

Use `@kiwa-test/core`'s `snapshotChain` / `revertChain` to isolate each test:

```typescript
import { dappE2eTest } from '@kiwa-test/core';
import { snapshotChain, revertChain, increaseTime } from '@kiwa-test/core';
import { expect } from '@playwright/test';
import type { Hex } from 'viem';

let snapshotId: Hex;

dappE2eTest.beforeEach(async ({ anvilPort }) => {
  // Snapshot the anvil state before each test
  const publicClient = makePublicClient(anvilPort);
  snapshotId = await snapshotChain(publicClient);
});

dappE2eTest.afterEach(async ({ anvilPort }) => {
  // Revert to the snapshot after each test (completely removes side effects)
  const publicClient = makePublicClient(anvilPort);
  await revertChain(publicClient, snapshotId);
});

dappE2eTest('TC-VESTING-001 release after the cliff passes', async ({ anvilPort }) => {
  const publicClient = makePublicClient(anvilPort);
  await increaseTime(publicClient, 30 * 24 * 60 * 60);  // advance 30 days
  // assert the vesting contract release
});

dappE2eTest('TC-VESTING-002 release stays at 0 before the cliff', async ({ anvilPort }) => {
  // The 30-day-advanced state from the test above is reverted; this starts before the cliff again
});
```

### Asserting a custom-error revert

```typescript
import { dappE2eTest } from '@kiwa-test/core';
import { expectCustomError } from '@kiwa-test/core';

dappE2eTest('TC-001 reverts with OnlyOwner', async ({ anvilPort }) => {
  const publicClient = makePublicClient(anvilPort);
  const attacker = createWalletClient({ /* attacker account */ });

  try {
    await attacker.writeContract({
      address: nftAddr, abi: nftArtifact.abi,
      functionName: 'mint', args: [attacker.account.address],
    });
    throw new Error('should have reverted');
  } catch (error) {
    // Assert the custom error name by walking viem's BaseError chain
    expectCustomError(error, 'OnlyOwner');
  }
});
```

## Step 4 — Add test scripts to `package.json`

```json
{
  "scripts": {
    "test": "node --import tsx tests/prepare-env.ts && playwright test",
    "test:headed": "node --import tsx tests/prepare-env.ts && playwright test --headed"
  }
}
```

You can also wire `prepare-env.ts` through Playwright's `globalSetup` (standard Playwright style):

```typescript
// playwright.config.ts
export default defineConfig({
  globalSetup: './tests/prepare-env.ts',
  // ...
});
```

## Step 5 — Run it + check for flakes

```bash
# Single run
pnpm test

# 4 consecutive rounds for flaky detection
for r in 1 2 3 4; do echo "=== Round $r ==="; pnpm test 2>&1 | tail -3; done
```

## `@kiwa-test/core` quick reference for the main APIs

Details: [docs/en/api/README.md](../../docs/en/api/README.md).

| Function | Role | Usage |
|---|---|---|
| `dappE2eTest` | Extended Playwright `test` fixture (injects `anvilPort`) | `dappE2eTest('TC-001 ...', async ({ anvilPort, page }) => {...})` |
| `startAnvil` | Spawn an anvil child process | `const handle = await startAnvil({ port })` |
| `startAnvilCluster` | Launch multiple anvils for different chain IDs at once | `const cluster = await startAnvilCluster({ chains: [...] })` |
| `startAnvilFork` | Mainnet / Sepolia fork | `startAnvilFork({ forkUrl, blockNumber, port })` |
| `loadForgeArtifact` | Read `forge-out/X.sol/X.json` and extract ABI / bytecode | `const { abi, bytecode } = loadForgeArtifact({ path })` |
| `runE2EPrepareEnv` | One function for anvil startup + deploy callback + `.env.local` write | `await runE2EPrepareEnv({ port, deploy: async ({ wallet }) => {...} })` |
| `waitForChainState` | Predicate-based polling for contract view state | `await waitForChainState({ client, predicate: () => ..., timeout })` |
| `snapshotChain` / `revertChain` | anvil state snapshot / revert | Use in `beforeEach` / `afterEach` for test isolation |
| `increaseTime` / `mineBlock` / `setNextBlockTimestamp` | Time control | For vesting / TTL / timelock tests |
| `impersonateAccount` / `setBalance` | Impersonate any address / inject balance | `impersonateAccount(client, addr)` |
| `expectEvent` | Extract and assert events from a tx receipt | `expectEvent(receipt, abi, 'Transfer', [from, to, 1n])` |
| `expectCustomError` | Assert a custom error name by walking viem's BaseError chain | `expectCustomError(error, 'OnlyOwner')` |
| `expectBalanceChange` / `expectEthBalanceChange` | hardhat-chai-matchers-compatible balance-delta assertions | `expectEthBalanceChange(client, addr, parseEther('1'), action)` |
| `getFreePort` | Ask the OS for a free port | Avoid port collisions |

## How to choose between direct library usage and the skill path

| Perspective | Direct library usage | Skill path |
|---|---|---|
| Test design | You hand-write the tests and decide the viewpoints / case count yourself | The skills auto-design them with the 11-viewpoint catalog + 5-axis review |
| Spec document | Optional (write it yourself only if you want it) | `tests/spec/{layer}/test-spec-X.md` is generated automatically |
| 100% coverage | You add tests manually | Auto loop drives coverage to 100% or a justified impossible decision |
| Quality review | Manual PR review by eye | `spec-review` / `test-review` / `result-review` run automatically |
| Learning cost | Low (enough if you already know Playwright + viem) | You need to understand the skill contracts |
| Typical use | Add tests into an existing suite / reuse only some fixtures | Build a large test suite from zero to OSS-publication quality |

## Troubleshooting

| Symptom | Fix |
|---|---|
| `Cannot find module '@kiwa-test/core'` | Install with `pnpm add -D @kiwa-test/core`, and build with `pnpm -F @kiwa-test/core build` in a monorepo |
| `ReferenceError: require is not defined` | Add `"type": "module"` to `package.json` |
| `Executable doesn't exist .../chrome-headless-shell` | `pnpm exec playwright install chromium` |
| anvil port collision (`EADDRINUSE: 8545`) | Stop the existing daemon with `pkill -f anvil` or use a dynamic port via `getFreePort()` |
| `forge-out/*.json` not found | Run `forge build` to generate artifacts |
| `process.env.NEXT_PUBLIC_X` is `undefined` | Include the key in the `return` value of the deploy callback in `prepare-env.ts`, or load `.env.local` via `dotenv.config()` |
